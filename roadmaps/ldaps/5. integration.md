### âœ… **A. Proses Login LDAP (Bind User)**

1. **User masukkan username dan password**
2. Aplikasi **bind ke LDAP** menggunakan DN user (misal: `uid=alwi,ou=users,dc=example,dc=com`)
3. Jika bind berhasil â†’ login sukses
4. Aplikasi bisa lanjut ambil data user (email, grup, dsb)

### ðŸ”§ **1. Integrasi LDAP di Jenkins**

Jenkins â†’ **Manage Jenkins â†’ Configure Global Security â†’ LDAP**

* **Server:** `ldap://localhost:389`
* **Root DN:** `dc=example,dc=com`
* **User Search Base:** `ou=users`
* **User Search Filter:** `(uid={0})`
* **Manager DN:** `cn=admin,dc=example,dc=com`
* **Manager Password:** `admin`

Jenkins akan:

* Mencari user berdasarkan uid
* Bind pakai DN dan password user â†’ jika sukses = login

### ðŸ”§ **2. Integrasi LDAP di GitLab**

Edit file `gitlab.rb`:

```ruby
gitlab_rails['ldap_enabled'] = true
gitlab_rails['ldap_servers'] = {
  'main' => {
    'label' => 'LDAP',
    'host' => 'localhost',
    'port' => 389,
    'uid' => 'uid',
    'bind_dn' => 'cn=admin,dc=example,dc=com',
    'password' => 'admin',
    'base' => 'dc=example,dc=com'
  }
}
```

Lalu jalankan:

```bash
sudo gitlab-ctl reconfigure
```

### ðŸ”§ **3. Integrasi LDAP di Nextcloud**

* Masuk sebagai admin â†’ Settings â†’ LDAP/AD Integration
* Masukkan:

  * **Server:** `ldap://localhost`
  * **User DN:** `cn=admin,dc=example,dc=com`
  * **Password:** `admin`
  * **Base DN:** `dc=example,dc=com`
  * **Login filter:** `uid=%uid`

## ðŸ§ª Tips Bind Login

Untuk melakukan bind secara manual dari CLI:

```bash

  -D "uid=alwi,ou=users,dc=example,dc=com" \
  -w alwi123 \
  -H ldap://localhost
```

Kalau berhasil â†’ akan muncul `dn:uid=alwi,ou=users,dc=example,dc=com`
